**DeepDungeon** 是一个纯控制台版本的硬核 Roguelike 地牢游戏，采用 C++17 开发。

- ✅ 程序化地图生成

- ✅ 回合制战斗系统

- ❌ 图形用户界面 (GUI)
- ❌ 网络多人游戏
- ❌ 存档/读档系统 (计划中)
- ❌ 音效系统

## 2. 总体描述

### 2.1 产品愿景

DeepDungeon 旨在提供一个**轻量级、高难度、可重复游玩**的 Roguelike 地牢游戏，通过纯文本界面实现：

- **程序化生成**：每次游戏生成独特的地牢布局
- **永久死亡**：角色死亡后游戏结束，增加紧张感
- **回合制策略**：玩家每步需谨慎思考
- **探索与发现**：战争迷雾系统鼓励探索未知区域

### 2.2 用户特性

#### 2.2.1 目标用户群体

| 用户类型 | 描述 | 技术能力 |
|----------|------|----------|
| 核心玩家 | Roguelike 游戏爱好者，喜欢挑战和策略 | 高 |
| 学习开发者 | 学习 C++ 和游戏架构的学生 | 中-高 |
| 休闲玩家 | 偶尔游玩的普通用户 | 低-中 |

#### 2.2.2 用户角色 (Actor)

**主要角色：玩家 (Player)**

- 通过键盘输入控制角色移动和动作
- 需要管理生命值、药水数量和装备
- 目标：消灭所有怪物或找到出口

#### 3.2.2 详细功能描述

##### FR-01: 地图生成

| 属性 | 描述 |
|------|------|
| 需求ID | FR-01 |
| 优先级 | 高 |
| 描述 | 系统在游戏启动时自动生成随机地牢地图 |
| 输入 | 随机种子 (seed) |
| 处理 | 使用程序化算法生成房间和走廊 |
| 输出 | 完整的地图数据结构 |
| 约束 | 地图尺寸固定为 40x20 |

**业务规则：**
- 地图必须包含至少 1 个房间
- 房间之间必须有走廊连接
- 必须生成怪物、武器和药水

##### FR-02: 玩家移动

| 属性 | 描述 |
|------|------|
| 需求ID | FR-02 |
| 优先级 | 高 |
| 描述 | 玩家通过键盘输入控制角色在地牢中移动 |
| 输入 | 键盘按键 (W/A/S/D) |
| 处理 | 验证移动合法性，更新玩家位置 |
| 输出 | 玩家新位置，更新后的游戏画面 |

**业务规则：**
- 不能移动到墙壁位置
- 不能移动到水位置 (但水是可站立的)
- 移动后触发 FOV 更新
- 移动后检查是否有物品可拾取
- 移动时若遇到怪物则触发战斗

##### FR-03: 战争迷雾 (FOV)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-03 |
| 优先级 | 高 |
| 描述 | 系统根据玩家位置计算可见区域，隐藏未探索区域 |
| 输入 | 玩家当前位置 |
| 处理 | 基于 FOV_RADIUS (默认 5) 计算可见图块 |
| 输出 | 更新地图的 visible 和 explored 标记 |

**业务规则：**
- 可见范围半径默认为 5 个图块
- 已探索但未可见的图块显示为灰色
- 未探索的图块完全隐藏

##### FR-04: 战斗系统

| 属性 | 描述 |
|------|------|
| 需求ID | FR-04 |
| 优先级 | 高 |
| 描述 | 玩家与怪物进行回合制战斗 |
| 输入 | 玩家攻击指令 (移动到怪物位置) |
| 处理 | 计算伤害，更新双方 HP，检查存活状态 |
| 输出 | 战斗结果，经验值奖励 |

**伤害计算公式：**
```
damage = max(1, attacker.atk - defender.def)
```

**业务规则：**
- 玩家攻击后怪物立即反击
- 怪物 HP 降为 0 时死亡，玩家获得经验值
- 玩家 HP 降为 0 时游戏失败

##### FR-05: 物品拾取

| 属性 | 描述 |
|------|------|
| 需求ID | FR-05 |
| 优先级 | 高 |
| 描述 | 玩家移动到物品位置时自动拾取 |
| 输入 | 玩家移动指令 |
| 处理 | 检测玩家位置是否有物品，应用物品效果 |
| 输出 | 物品效果应用，物品从地图移除 |

**物品类型：**

| 物品类型 | 符号 | 效果 |
|----------|------|------|
| 武器 | `/` | 增加攻击力 (+5) |
| 药水 | `!` | 增加药水库存 (+1) |

##### FR-06: 怪物 AI

| 属性 | 描述 |
|------|------|
| 需求ID | FR-06 |
| 优先级 | 高 |
| 描述 | 怪物每回合自动追踪并攻击玩家 |
| 输入 | 玩家当前位置 |
| 处理 | 怪物向玩家方向移动，若相邻则攻击 |
| 输出 | 怪物新位置或玩家受到伤害 |

**业务规则：**
- 怪物只能看到可见范围内的玩家
- 怪物每回合移动一步
- 怪物与玩家相邻时自动攻击

##### FR-07: 药水使用

| 属性 | 描述 |
|------|------|
| 需求ID | FR-07 |
| 优先级 | 中 |
| 描述 | 玩家使用药水恢复生命值 |
| 输入 | 键盘按键 (P) |
| 处理 | 检查药水库存，恢复 HP |
| 输出 | 玩家新 HP 值 |

**业务规则：**
- 必须有药水库存才能使用
- 恢复量固定 (待定义)
- HP 不能超过 maxHp

##### FR-08: 状态查看

| 属性 | 描述 |
|------|------|
| 需求ID | FR-08 |
| 优先级 | 中 |
| 描述 | 显示玩家当前状态信息 |
| 输入 | 自动触发 (每回合渲染) |
| 处理 | 格式化玩家属性 |
| 输出 | 状态栏文本 |

**状态信息包括：**
- HP (当前/最大)
- ATK (攻击力)
- DEF (防御力)
- Exp (经验值)
- Level (等级)
- Potion (药水数量)

##### FR-09: 升级系统

| 属性 | 描述 |
|------|------|
| 需求ID | FR-09 |
| 优先级 | 中 |
| 描述 | 玩家获得足够经验后升级 |
| 输入 | 经验值增加 |

| 输出 | 玩家等级和属性提升 |

**业务规则：**
- 升级阈值：50 Exp
